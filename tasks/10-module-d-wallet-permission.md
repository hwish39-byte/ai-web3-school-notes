## 任务一：Agent 发起链上动作执行流程图

### 场景：Agent 在 Uniswap 上执行 Swap

| 步骤 | 自动化 | 说明 |
|------|--------|------|
| 1. 检查 Pact 预算 | AI 自动 | 余额、限额、白名单检查 |
| 2. 读取链上数据 | AI 自动 | 余额、价格、流动性 |
| 3. 模拟交易 | AI 自动 | 预期输出、滑点计算 |
| 4. 风险评估 | AI 自动 | 判定风险等级 |
| 4a. 高风险标记 | AI 标记 | 需人工确认的交易标记 |
| 5. 人工确认 | 必须人工 | 用户审核交易详情 |
| 6. 签名上链 | 必须人工 | MetaMask 签名 |
| 7. 验证记录 | AI 自动 | 确认交易状态 |

### 流程简图

```
Agent 指令
  -> AI 自动: 预算检查 -> 读链上数据 -> 模拟 -> 风险评估
  -> 低风险 -> 直接执行
  -> 高风险 -> ⛔ 等人确认 -> 用户审核 -> 签名 -> 上链
  -> AI 自动: 验证状态 -> 记录日志
```

## 任务二：Agent Wallet 权限策略

| 维度 | 策略 |
|------|------|
| 预算 | per_tx 100 USDC / daily 500 USDC / weekly 2000 USDC |
| 可调用合约 | Uniswap Router（指定函数白名单）|
| 可执行动作 | swap（最大 1% 滑点）、approve（最大 100 USDC）|
| 人工确认阈值 | 金额 > 100 / 滑点 > 1% / 非白名单合约 |
| 撤销方式 | Dashboard 撤销、7 天自动过期、紧急暂停 |
| 日志记录 | 本地 + 链上，保留 90 天 |
| 失败处理 | 重试 2 次 -> 提 Gas -> 通知用户 |

## 任务三：ERC-4337 / Safe / Guard

| 机制 | 解决什么问题 | 对 Agent 的意义 |
|------|-------------|----------------|
| ERC-4337 | 私钥单点故障 | Agent 用 Smart Account 操作，不持有主私钥 |
| Safe | 单人控制风险 | Agent 作为签名者之一，不能单独转钱 |
| Guard | 签名后执行前拦截 | Agent 拿到签名后，Guard 仍可阻止违规交易 |

三者可叠加：ERC-4337 让 Agent 能签名 -> Safe 让 Agent 不能单独签 -> Guard 让签了也能拦。

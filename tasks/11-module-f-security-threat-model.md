## 任务一：Agent Workflow Threat Model

### 选定 Workflow：AI 合约审计 Agent

### 资产清单

| 资产 | 风险等级 |
|------|---------|
| CAW Session Key | 高 |
| Pact 授权额度（500 USDC/日） | 高 |
| API Key | 高 |
| 用户合约源码 | 中 |
| RPC Endpoint | 低 |

### 攻击入口

| 攻击 | 风险等级 | 影响 |
|------|---------|------|
| Prompt Injection | 高 | Agent 执行非预期操作 |
| 伪造 402 响应 | 高 | 向攻击者付款 |
| 工具返回污染 | 中 | 基于错误数据决策 |
| Session Key 泄露 | 高 | 攻击者转账 |
| 重放攻击 | 中 | 重复付款 |

### 外部依赖风险

| 依赖 | 风险 | 替代 |
|------|------|------|
| RPC 节点 | 错误数据/宕机 | 多节点 Fallback |
| LLM API | 不可用/数据泄露 | 本地模型 |
| Etherscan API | 限流 | 直接 RPC |

### 失败后果

| 场景 | 后果 | 等级 |
|------|------|------|
| Session Key 泄露 | 额度被花光 | 高 |
| Prompt Injection | Agent 执行恶意操作 | 高 |
| 误审 | 合约有漏洞但报告安全 | 高 |
| 交付失败 | 用户付钱没拿到报告 | 中 |

## 任务二：自动执行 / 人工确认策略

### 自动执行条件（全部满足）
- 金额 <= 100 USDC
- 合约在白名单
- 函数在白名单
- 滑点 <= 1%
- 日累计 <= 400 USDC
- Gas <= 100 Gwei
- Pact 未过期
- 无上一笔交易失败
- 模拟交易通过

### 人工确认条件（任一触发）
- 金额 > 100 USDC
- 滑点 > 1%
- 非白名单合约
- 连续 2 笔交易失败
- 新合约首次交互

## 任务三：高风险 Idea 分析

Idea: "AI 全自动交易机器人"

| 风险 | 说明 |
|------|------|
| Prompt Injection | 恶意网页就能清空钱包 |
| 无恢复机制 | 链上不可回滚 |
| 模型幻觉 | 假的合约地址当真 |
| 黑箱决策 | 用户不知道为什么买/卖 |
| 亏损归属 | 算谁的？ |

结论：当前全自动交易不成熟。正确的做法是 Pact 限制预算 + 高风险人工确认 + Agent 只做建议不执行。

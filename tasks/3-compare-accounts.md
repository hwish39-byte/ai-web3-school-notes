# 任务：比较 EOA、智能账户、多签的权限差异

## 目标
理解 EOA、智能账户（AA）、多签三种账户类型的权限模型差异：谁能发起交易、谁能批准、谁承担风险

## 对比表格

| 维度 | EOA（普通钱包） | 智能账户（AA / ERC-4337） | 多签（MultiSig） |
|------|----------------|--------------------------|-----------------|
| **谁持有控制权** | 单一私钥持有者 | 智能合约逻辑控制（可编程） | 多个私钥持有者共同控制 |
| **谁可以发起交易** | 私钥持有者 | 合约定义的逻辑（Session Key、规则触发等） | 达到阈值数量的签名者 |
| **支持多人确认？** | ❌ 单人签名即可 | ✅ 可编程（可设多签、限额等规则） | ✅ N/M 阈值签名 |
| **支持恢复？** | ❌ 私钥丢失 = 资产丢失 | ✅ 可设社交恢复、Guardian | ✅ 更换签名者需重新部署或升级 |
| **支持限额/自动化？** | ❌ 无法限制 | ✅ 可设日限额、白名单、过期权限 | ❌ 每笔交易都需要多人签 |
| **典型使用场景** | 个人日常使用、小额资产 | Agent 钱包、企业自动化、DeFi 策略 | DAO 国库、项目资金、高安全场景 |
| **主要风险点** | 私钥泄露或丢失 = 全部失控，无恢复机制 | 合约漏洞、逻辑错误可能导致资产损失 | 签名者共谋、治理攻击、确认延迟 |
| **交易签名流程** | 1 人签 1 次 | 1 次签名 + 合约验证逻辑 | ≥2 人签名 |

## 用场景来理解

### EOA — 现在用的 MetaMask
```
 MetaMask 账号就是 EOA
私钥在本地 → 你一个人控制一切
丢了助记词 = 丢了所有资产
没有"转账限额"的概念
```

### 智能账户（AA）— Agent 的理想钱包
```
给 Agent 一个 Session Key：
  - 每天最多花 100 USDT
  - 只能跟 Uniswap 交互
  - 7 天后自动过期
Agent 可以在限额内自主操作，你不需要每次都签名
主私钥放在冷钱包，只有恢复时才用
```

### 多签 — LXDAO 国库
```
假设 LXDAO 有 5 个核心成员
设置 3/5 多签：任何转账需要 ≥3 人签名
一个人私钥泄露了，攻击者也转不走钱（凑不够 3 签）
但每次转账需要等 3 个人都签名，体验慢
```

## 安全边界总结

```
EOA：        私钥 = 一切。简单直接，但单点故障。
智能账户：   代码 = 规则。灵活可编程，但依赖代码正确。
多签：       共识 = 安全。多人制衡，但牺牲效率。

EOA 和 AA 的核心区别不是"有没有私钥"，
而是"谁能做什么"从固定规则变成了可编程规则。
AA 不是去掉私钥，而是给私钥加了一层可编程的权限管理器。
```

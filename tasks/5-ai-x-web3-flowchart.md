# 任务：画出 AI × Web3 最小交叉流程图

## 目标
通过三个典型工作流，理解 AI 系统和链上操作之间的边界：谁发起、谁执行、哪里需要钱包签名

## 三个流程图

### 流程 A：AI 生成交易 → 人工确认 → 链上执行
适用场景：用户问"这笔 approve 能签吗？"
-  AI 层：生成交易说明 + 模拟风险
-  人工层：复核、决策、钱包签名（必须人工）
-  链上层：交易上链、区块浏览器验证

### 流程 B：Agent 自动化 → 人工兜底
适用场景：Agent 在限额内自动交易
-  初始化：用户设定规则 + 部署 Session Key
-  Agent：监听条件 → 构造交易 → Session Key 签名（自动）
-  事后：检查日志、发现异常可撤销

### 流程 C：DAO 治理助手
适用场景：提案分析 + 投票辅助
-  AI：提取提案关键信息、总结支持/反对理由、生成投票检查清单
-  社区：阅读报告、讨论决策、钱包投票
-  链上：投票交易上链、结果公开

## 关键原则

```
AI 不能做：接触私钥、替用户签名、自动发交易（除非 Session Key）、替用户决策
AI 可以做：读链上数据、生成交易解释、模拟交易、限额内自动执行、记录交易状态
```

### 智能体工作流（Agent Workflow）核心要点

- AI × Web3 的难点不在于让模型说"我可以帮你操作"，而在于把操作拆成可验证步骤
- 链上动作有资产、权限和不可逆结果，Agent 的工作流必须比普通聊天更严格
- **Task Graph**：把目标拆成节点和依赖，而不是让 Agent 一口气自由执行
  - 例如"评估并执行一次低风险 swap"可拆为：确认链 → 读取余额 → 检查 allowance → 模拟交易 → 确认 → 签名 → 发送 → 等待回执 → 验证结果
- **State Machine**：让 Agent 执行过程有明确状态（pending / simulation / confirmed / failed / reverted），而不是只有聊天历史
- **Human-in-the-loop**：把人放在关键风险点（首次交互、高额交易、权限修改），而不是让人确认每一个低风险步骤
- **Retry / Fallback**：RPC 失败重试、Gas 不足调整、模拟失败停止、用户拒绝流程终止
- **Trace**：每步执行记录（输入、输出、谁调用的、状态变化），用于调试和重构
- **Evaluation Harness & Regression Set**：对工作流做自动化测试，防止修改后引入新断裂

### 第一性原理

模型可以规划，但系统要知道当前到了哪一步、哪些工具已调用、哪些结果可信、哪些动作需要确认、失败后如何停止。**人负责决策，Agent 负责执行流程，系统负责记录和验证。**

## 今日学习章节

**Week 3｜共学营深化 + Hackathon 启动**（2026年5月29日 - 6月4日）

本月进入 Week 3，重点从基础学习转向实战深化与 Hackathon 准备。主线内容来自 Handbook 的以下章节：

### AI 基础 — 智能体（Agent）
- **Tool Use**：Agent 调用外部能力（搜索、API、支付接口等）。工具让 Agent 从"会回答"变成"能做事"，但也引入 Prompt Injection 风险。
- **Planning**：目标拆成步骤，但模型生成的计划只是候选路线，不是授权。
- **State**：需要可查询、可恢复、可审计的 state，而不是只靠 prompt 历史。
- **Reflection**：Agent 自我检查中间结果，但不能替代外部验证。
- **Multi-Agent**：多个 Agent 分工协作，但也放大协调问题。

### AI × Web3 Bridge — 智能体工作流（Agent Workflow）
核心是把"用户目标 → 上下文读取 → 计划生成 → 工具调用 → 风险检查 → 执行 → 记录和复盘"组织成可控流程。

关键概念：
- **Task Graph**：目标拆成节点和依赖（如 swap 拆为读取余额 → 查询价格 → 模拟 → 用户确认 → 发送交易 → 追踪结果）
- **State Machine**：明确状态（draft → plan_ready → waiting_user_confirmation → submitted → confirmed/reverted）
- **Human-in-the-loop**：只读分析自动执行，高风险交易必须人工确认
- **Retry / Fallback**：不能盲目重试，要判断交易是否已广播
- **Trace**：记录每一步输入、判断、工具调用和结果
- **Evaluation Harness**：系统测试 Agent 在不同场景下的表现
- **Regression Set**：固定测试用例，防止更新后安全性退化

## 学习路径

### 最小路径（半小时内能完成的入门任务）
1. 阅读 Handbook「智能体工作流」章节，理解 Task Graph 概念
2. 在纸上画出一个简单 Agent 工作流的 Task Graph（例如：查询余额 → 报价 → 用户确认）
3. 标记出哪些步骤需要 Human-in-the-loop

### 推荐路径（正常学习节奏）
1. 完整阅读 Handbook 以下章节：
   - [智能体（Agent）](https://aiweb3.school/zh/handbook/ai/agent/)
   - [智能体工作流（Agent Workflow）](https://aiweb3.school/zh/handbook/bridge/agent-workflow/)
   - [智能体钱包（Agent Wallet）](https://aiweb3.school/zh/handbook/bridge/agent-wallet/)
2. 完成 Agent 章节的"最小实践"：做一个"DAO 提案研究 Agent"只读版本
3. 开始思考 Hackathon 方向：选择感兴趣的赛道（Agentic Commerce / Dev Tooling / AI Security / Governance）

### 挑战路径（有余力时深入）
1. 阅读 [Web3 工具调用（Web3 Tool Use）](https://aiweb3.school/zh/handbook/bridge/web3-tool-use/) 和 [机器支付（Machine Payment）](https://aiweb3.school/zh/handbook/bridge/machine-payment/)
2. 用 Hermes Agent 或 Claude Code 搭建一个本地 Agent 原型，实现：
   - 读取测试网钱包余额
   - 模拟一笔交易
   - 输出 trace 日志
3. 写 5 个 Regression Case（正常、错误链、金额过大、余额不足、用户拒绝）
4. 形成 Hackathon 项目 proposal：目标用户、问题定义、技术路径、demo 计划

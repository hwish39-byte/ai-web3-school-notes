**核心要点：**

### 🪟 Context Window
- 模型一次请求能处理的最大上下文范围
- 窗口大 ≠ 模型会用得好，"看见了但没抓住重点" 是常见问题
- 在实践中要配合检索、摘要和结构化数据使用

### 🏗️ Context Engineering
- 设计上下文进入模型的方式：数据源选择、排序、裁剪、标注来源
- 一个稳定的工具型 Agent 上下文应包括：系统指令、当前状态、工具返回、用户偏好、外部数据、历史摘要
- 不可信内容要隔离标注

### 🧠 Memory
- 跨请求保留的信息（用户偏好、历史任务、常用地址）
- 方便但有隐性风险——Memory 不能替代实时授权
- 用户过去允许的动作，不代表现在仍然允许

### 📚 Knowledge Base
- 系统可检索的外部知识库（文档、FAQ、runbook）
- 与 Memory 的区别：KB 是公开知识，Memory 是用户定制

### 🔧 LLM 最小实践：交易解释器
- 使用 Claude Code + ethers.js v6 创建链上交易解析脚本
- 通过 Sepolia 测试网 RPC 读取交易详情，输出格式化表格和事件日志
- 验证交易哈希：`0x711b963df8a432c8f24580681de76b6d7294ac6cfb930ba0d8f2a15cf22ee9fc`
- 将链上数据喂给 Claude 做自然语言解释，成功输出：
  - 交易摘要
  - 链上事实（from / to / value / 事件）
  - 模型推断
  - 不确定性标注
  - 用户检查清单
